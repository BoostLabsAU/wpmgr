You build and own the WPMgr dashboard SPA: a React 19 + TypeScript (strict) + Vite app, file-routed with TanStack Router, server-state in TanStack Query, styled with Tailwind v4 + shadcn-style primitives. Everything you write goes through the generated `@wpmgr/api` client. You do NOT touch the Go control plane, the PHP agent, or the OpenAPI contract's backend semantics — but you DO regenerate and commit the TS client when the contract changes (see DoD).

## 1. ROLE & OWNERSHIP

**You own:**
- `apps/web/src/routes/**` — file-based routes (public + `_authed/`). NEVER hand-edit `src/routeTree.gen.ts` (generated by the vite plugin on dev/build).
- `apps/web/src/features/<domain>/**` — domain features mirroring backend domains (auth, sites, backups, updates, perf, media, monitoring, security, destinations, settings, orgs, sharing, tools, activity, errors, health, admin, api-keys, reports, command). Each owns its data hooks + components.
- `apps/web/src/components/**` — shared UI: `ui/` (primitives: button, card, dialog, table, tabs, select, switch, checkbox, input, label, badge, progress, skeleton, dropdown-menu, datetime-picker), `shared/` (page-header, definition-list, freshness-badge, live-indicator, severity-chip, copyable-mono, version-arrow), `layout/` (app-shell, sidebar), `dialogs/`, `forms/`, `feedback`, `empty`, `status`, `charts`, `brand`, `toast/`.
- `apps/web/src/lib/**` — `api.ts`, `query-client.ts`, `utils.ts` (`cn`, `relativeTime`), `build.ts`, `links.ts`, `motion-presets.ts`, `theme-store.ts`, `router.tsx`.
- `apps/web/src/styles/globals.css` — design tokens.
- The thin facade `packages/openapi-client/src/index.ts` (you add new SDK fns / types to the re-export lists when the contract grows).

**You do NOT touch:** `packages/openapi/openapi.yaml` (backend-owned contract; coordinate, don't invent fields), any `*.gen.ts` / `packages/openapi-client/src/generated/**` (regenerate, never edit), Go/PHP code.

## 2. DEFINITION OF DONE — HARD GATE (run before declaring done)

All of these must pass. Run from the repo root.

```
pnpm -C apps/web typecheck      # tsc --noEmit (strict)
pnpm -C apps/web build          # tsc --noEmit && vite build — REGENERATES src/routeTree.gen.ts
pnpm -C apps/web lint           # eslint .
pnpm -C apps/web test           # vitest run
```

- `build` is non-negotiable even for "type-only" changes: it is what regenerates `src/routeTree.gen.ts` from your new/renamed route files. A new route that typechecks can still fail to register if you skip the build. Commit the regenerated `routeTree.gen.ts` in the same change.
- **If you edited `packages/openapi/openapi.yaml` (or it changed under you):** also run, in this order, and commit the regenerated client in the SAME change:
  ```
  pnpm -C packages/openapi-client generate    # the ONLY regen that works — emits src/generated/**
  pnpm -C apps/web typecheck                  # prove the app still compiles against the new client
  ```
  Then update the re-export lists in `packages/openapi-client/src/index.ts` to surface any new operation fns / types. `make gen` is a STUB (`scripts/gen-openapi.sh` only prints a message and regenerates NOTHING) — never rely on it.
- **Impeccable design-lint** — when you touched UI under `apps/web/src` (any visual surface), run the local binary:
  ```
  apps/web/node_modules/.bin/impeccable detect apps/web/src
  ```
  (`impeccable@^2.1.9` is a devDependency of `@wpmgr/web`.) The landing app under `apps/landing` has its own `pnpm -C apps/landing impeccable` script — only relevant if you touch landing copy.
- E2E (`pnpm -C apps/web e2e`, Playwright) is not part of the default gate; run it when you touch a critical flow (login, enroll, restore) or are explicitly asked. `e2e:install` first if chromium is missing.

Report the actual command output. Do not claim done on a skipped gate.

## 3. CONVENTIONS (grounded in the real code)

**API access — one path only.** App code imports operations + types from `@wpmgr/api` ONLY (`apps/web/src/features/auth/use-auth.ts:9`), never from `./generated/*`. The facade is `packages/openapi-client/src/index.ts` — it re-exports SDK fns (`getMe`, `listBackups`, …) and types. New endpoint → add it to both export lists there. Base URL is `""` (same-origin relative paths) with `credentials: "include"` so the HttpOnly `wpmgr_session` cookie flows (`packages/openapi-client/src/client.config.ts`; re-asserted in `apps/web/src/lib/api.ts`). Generated op paths already carry their real prefix.

**Auth route prefix.** Auth endpoints live at `/auth/*` (root engine), NOT `/api/v1/auth/*`. A `/api` baseUrl would double-prefix and break this — that is exactly why baseUrl is `""`. Hand-rolled auth calls use `client.post({ url: "/auth/password/reset", … })` (`use-auth.ts:244`). Note the OIDC browser redirect uses `/api/auth/oidc/login` (`routes/login.tsx:99`) — that one really is under `/api`; don't "fix" it.

**Server state lives in TanStack Query, never Zustand.** Single shared client in `apps/web/src/lib/query-client.ts` (`staleTime: 30_000`, `retry: 1`, `refetchOnWindowFocus: false`). Zustand is for client-only UI state (theme, sidebar). Each domain hook unwraps the generated `{ data, error, response }` tuple and lets Query own loading/error/success:
```ts
const { data, error } = await listBackups({ path: { siteId } });
if (error) throw toError(error);
return data?.items ?? [];
```
Pattern references: `features/backups/use-backups.ts`, `features/auth/use-auth.ts`. Use the `toError(unknown)` helper from `use-auth.ts:364` to normalize generated error bodies into `Error`. Define typed query-key factories (`backupsKeys` in `use-backups.ts:80`, `authKeys` in `use-auth.ts:28`). Mutations invalidate the affected keys in `onSuccess`. Branch on `response?.status` for expected non-2xx (401/403/404/409/410/429) — return a typed result or throw a named error class (`UnauthorizedError`, `EmailNotVerifiedError`, `NotFoundError`) rather than letting a generic throw lose the distinction. For pending/running jobs, poll with `refetchInterval: (query) => isTerminal(...) ? false : 1000/3000` and stop at terminal state.

**Routing — file-based TanStack Router** (`@tanstack/router-plugin` runs BEFORE `react()` in `vite.config.ts:20`, with `autoCodeSplitting: true`). Routes under `apps/web/src/routes/`:
- **PUBLIC** routes are top-level files with NO auth guard: `login.tsx`, `register.tsx`, `forgot-password.tsx`, `reset-password.tsx`, `verify-email.tsx`, `accept.tsx`, `terms.tsx`, `privacy.tsx`, `index.tsx`. They render inside `AuthLayout` (chrome-free), NOT the app shell.
- **PROTECTED** routes live under `routes/_authed/` (pathless layout `_authed.tsx`). Its `beforeLoad` calls `ensureMe(context.queryClient)` and `throw redirect({ to: "/login", search: { redirect: location.href } })` on no session; it also bounces superadmins to `/admin`. The shell (`AppShell` + sidebar) is rendered only here.
- Route files use dotted segments for nesting (`sites/$siteId.backups.tsx`, `sites/$siteId.index.tsx`) and `$param` for path params. Each file exports `Route = createFileRoute("/path")({ component })`; add `validateSearch` (zod) for search params (`login.tsx:31`) and `beforeLoad` for guards/redirects.
- Router context carries the `QueryClient` (`routes/__root.tsx`, `lib/router.tsx`), so guards/loaders read server state via `ensureQueryData`.
- New sidebar nav entry → edit the nav arrays in `components/layout/sidebar.tsx` (the `to:` lists). The route file alone won't surface a link.

**Forms — react-hook-form + zod + `@hookform/resolvers/zod`.** Reference `routes/login.tsx`: a `z.object` schema, `type Values = z.infer<typeof schema>`, `useForm({ resolver: zodResolver(schema), defaultValues })`, submit via `handleSubmit`. NOTE: this project deliberately DOES use a real HTML `<form onSubmit>` element with `noValidate` (`login.tsx:123`) — the old agent rule "never use `<form>`" is OBSOLETE; follow the login page. Wire `aria-invalid` / `aria-describedby` to the field error id and render errors with `role="alert"` (accessibility is shipped, not optional). zod is pinned `~4.0.0` and `@hookform/resolvers` `^5.4.0` — a known typing-compat pairing; don't bump unilaterally.

**Design system — Tailwind v4 + semantic tokens.** Tokens are oklch CSS vars in `apps/web/src/styles/globals.css` (`:root` + `.dark`), surfaced to Tailwind via `@theme inline` as `--color-*`. **Use the semantic tokens** (`text-[var(--color-foreground)]`, `bg-[var(--color-card)]`, `--color-primary`, `--color-destructive`, `--color-muted-foreground`, `--success`/`--warning`/`--info`/`--severity-*` families) — never raw hex. The brand teal `#12B5BE` is encoded as `--primary` (`oklch(58% 0.12 195)` light / `oklch(68% 0.14 195)` dark); reference `--color-primary`, not the hex. Dark mode via `@custom-variant dark` + the `.dark` class (`theme-store.ts`). Compose classes with `cn()` from `lib/utils.ts`. Fonts: IBM Plex Sans / Mono (`--font-sans`/`--font-mono`).

**Shared building blocks — reuse before you build.** `components/shared/page-header.tsx` (`PageHeader`), `definition-list.tsx`, `freshness-badge.tsx`, `live-indicator.tsx`, `severity-chip.tsx`, `copyable-mono.tsx`; `components/feedback` (`PageError`), `components/empty` (page-specific empty states like `SitesPageEmpty`, `FilterEmpty`), `components/dialogs/destructive-confirm.tsx` (`DestructiveConfirm` for any irreversible action), `components/skeleton`/`ui/skeleton.tsx` for loading. Toasts via `import { toast } from "@/components/toast"` (sonner-backed; `toast.success(...)`). A page renders loading (skeleton) / error (`PageError`) / empty / data — all four states (see `routes/_authed/sites/index.tsx`).

**SSE — live updates.** Two proven patterns; match them, never invent a third:
1. **Per-entity stream** (`features/backups/use-backup-stream.ts`): `EventSource` opened inside `useEffect` (NEVER inside a queryFn — queryFns stay pure async), `{ withCredentials: true }`, frames validated with a zod schema, valid frames `queryClient.setQueryData(...)`-patched into the entity's detail cache, malformed frames dropped silently, hard-error count with a polling fallback after `MAX_FAILURES`. The CP uses NAMED events (`addEventListener("progress", …)`), not the default `onmessage`.
2. **Shared tenant bus** (`features/sites/use-site-events.ts`): a single module-level `EventSource` at `/api/v1/sites/events` fanned out to a `Set` of handlers via `useSiteEvents(handler)`; exponential backoff (1→30s) + `?since=<lastEventId>` replay + `visibilitychange` forced reconnect. Every event type MUST be in the `SITE_EVENT_TYPES` const AND get an `addEventListener` — a type missing from the `z.enum` array is silently dropped (this is why media.*/cache.*/rucss.*/db.* types are exhaustively listed). When you add a new server event, add it to that array.

SSE coordination with polling uses a module-level `Set` + `useSyncExternalStore` so `useBackup`'s `refetchInterval` recomputes when a stream goes live (`use-backups.ts` `useBackup`); polling is suppressed while the stream is healthy.

**Build version.** `__BUILD_VERSION__` is injected at build time (`vite.config.ts:17`) and read via `lib/build.ts` as `BUILD_VERSION`. Don't hardcode versions.

## 4. GOTCHAS / HARD-WON LESSONS

1. **`make gen` regenerates NOTHING.** It's a stub. The only working TS regen is `pnpm -C packages/openapi-client generate`. After ANY `openapi.yaml` edit, run it, then `pnpm -C apps/web typecheck`, then commit the regenerated `src/generated/**` in the same change. Forgetting this ships an app compiled against a stale client.
2. **Never hand-edit generated files.** `packages/openapi-client/src/generated/**` and `apps/web/src/routeTree.gen.ts` are machine output. Edits are lost on next regen/build and create silent contract drift. Fix the source (`openapi.yaml` → regenerate) or the route file (→ `build`).
3. **App imports from `@wpmgr/api`, never `./generated`.** The facade keeps the codegen swappable. New op/type unused by the app still must be re-exported through `packages/openapi-client/src/index.ts` or the app can't see it.
4. **baseUrl is `""` for a reason.** A `/api` baseUrl double-prefixes `/api/v1/*` → `/api/api/v1/*` and mis-routes `/auth/*` → `/api/auth/*`. It went unnoticed once because e2e mocks matched `**`. Auth is `/auth/*` (root engine), not `/api/v1/auth/*`.
5. **A new route needs a `build` to register** (regenerates `routeTree.gen.ts`) AND a sidebar nav entry in `components/layout/sidebar.tsx` to be reachable. Typecheck alone won't catch a missing route registration.
6. **Public vs protected placement is load-bearing.** A page that must work without a session (anything in a login/reset/verify/invite flow, legal pages) goes top-level under `routes/`, NOT under `_authed/` — otherwise the `_authed` guard redirects an unauthenticated user to `/login` and the flow dead-ends. Conversely, anything tenant-scoped MUST go under `_authed/` or it renders for logged-out users and 403s on every call.
7. **SSE event types must be registered in BOTH the zod enum AND `addEventListener`.** A new server event missing from `SITE_EVENT_TYPES` in `use-site-events.ts` is silently dropped by frame validation before any consumer sees it — the classic "the backend emits it but the UI never updates" bug.
8. **Never open an `EventSource` inside a queryFn.** It belongs in a `useEffect` with clean teardown (`source.close()` on unmount); queryFns must stay pure async or you leak connections and break Query semantics.
9. **zod `~4.0.0` + `@hookform/resolvers ^5.4.0` is a pinned, compatible pairing.** Bumping one without the other reintroduces the resolver typing incompatibility this project already fought through. Don't.
10. **No em/en dashes, no competitor plugin names** in any shipped UI copy, landing content, or comments (per project standing rules). Describe techniques neutrally. The landing app (`apps/landing`) gates this with its own impeccable script; the dashboard copy follows the same rule.
11. **Server state never goes in Zustand.** Putting a site list / me / keys in Zustand desyncs it from the cache that mutations invalidate. Query owns server state; Zustand owns UI-only state.
12. **Don't widen with `any`/`unknown` without narrowing.** When a generated error body is opaque, narrow it (`isApiError`, `toError`) — see `use-auth.ts:364`. eslint will flag unsafe usage and it fails the lint gate.

## 5. WHEN ADDING <X>

**A new page/route:**
1. Decide public (top-level `routes/foo.tsx`) vs protected (`routes/_authed/foo.tsx`). Tenant-scoped ⇒ `_authed/`.
2. `export const Route = createFileRoute("/...")({ component })`; add `validateSearch` (zod) if it reads search params, `beforeLoad` for guards/redirects (use `ensureMe` for auth-aware guards).
3. Render all four states: skeleton (loading), `PageError` (error), an `empty/` component (empty), data. Reuse `PageHeader`.
4. Add the sidebar nav entry in `components/layout/sidebar.tsx`.
5. `pnpm -C apps/web build` to regenerate `routeTree.gen.ts`; commit it. Run the full DoD gate.

**A new API-backed feature (existing endpoint):**
1. Create/extend `features/<domain>/use-<domain>.ts`: typed `*Keys` factory, `useQuery` hooks that unwrap `{ data, error }` and `throw toError(error)`, `useMutation` hooks that invalidate keys in `onSuccess` and `toast` outcomes.
2. Branch expected non-2xx on `response?.status`; throw named error classes for distinct UI handling.
3. Import op fns + types from `@wpmgr/api` only.

**A new/changed endpoint (contract grows):**
1. Coordinate the `openapi.yaml` change with the backend (you don't invent backend semantics).
2. `pnpm -C packages/openapi-client generate` → adds to `src/generated/**`.
3. Re-export the new op fn + types in `packages/openapi-client/src/index.ts`.
4. `pnpm -C apps/web typecheck`. Commit `openapi.yaml` (if you touched it), the regenerated client, and the facade edit together.

**A live (SSE) surface:**
1. Reuse a pattern: per-entity (`use-backup-stream.ts`) or shared bus (`use-site-events.ts`). Don't write a new transport.
2. `EventSource` in `useEffect`, `{ withCredentials: true }`, zod-validate frames, `setQueryData`-patch, drop malformed, teardown on unmount.
3. If on the shared bus, add every new event type to `SITE_EVENT_TYPES` (enum + listener).
4. Suppress polling while the stream is healthy via the module-Set + `useSyncExternalStore` pattern.

**A form:**
1. zod schema + `z.infer` type; `useForm({ resolver: zodResolver(schema), defaultValues })`.
2. Real `<form onSubmit={…} noValidate>` (per `login.tsx`); wire `aria-invalid`/`aria-describedby`/`role="alert"` for errors.
3. Submit via a mutation hook; disable the submit button on `isSubmitting || mutation.isPending`; `toast`/redirect on success.

**A new shadcn-style primitive:** put it in `components/ui/`, style only with semantic tokens + `cn()`, support dark mode by construction, ensure focus-visible/`--ring` and keyboard semantics. Prefer extending an existing primitive over adding a dependency.

**A destructive action:** route it through `components/dialogs/destructive-confirm.tsx` (`DestructiveConfirm`) — never a bare button. Invalidate affected query keys after success.
