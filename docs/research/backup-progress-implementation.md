# Backup Progress UI — Implementation Dossier (React 19 + Tailwind v4 + shadcn/ui)

Companion to `backup-live-progress-ux.md`. This document answers the **engineering**
side: what components, libraries, and techniques actually deliver a smooth,
accessible, reactive progress experience in our stack.

Stack ground truth (`apps/web/package.json`):

- React 19 + TS strict + Vite 6
- Tailwind **v4** (`@tailwindcss/vite`, `@import "tailwindcss";` syntax)
- shadcn/ui already installed: Badge, Button, Card, Checkbox, Dropdown, Input,
  Label, Progress, Table (see `apps/web/src/components/ui/`)
- `lucide-react ^1.16` (yes, the legacy 0.x line — see note in §4)
- TanStack Query v5, TanStack Router file-based
- No Framer Motion, no Motion One, no react-spring, no `@number-flow/react` — we
  are CSS-only today and the existing `progress.tsx` uses `transition-[width]
  duration-300`.

---

## 1. shadcn/ui Progress component

**Yes, shadcn/ui ships one.** It is generated by
`npx shadcn@latest add progress` and wraps `@radix-ui/react-progress`. The
generated file is ~22 lines and looks like:

```tsx
import * as ProgressPrimitive from "@radix-ui/react-progress";
import { cn } from "@/lib/utils";

function Progress({ className, value, ...props }: React.ComponentProps<typeof ProgressPrimitive.Root>) {
  return (
    <ProgressPrimitive.Root
      data-slot="progress"
      className={cn("bg-primary/20 relative h-2 w-full overflow-hidden rounded-full", className)}
      {...props}>
      <ProgressPrimitive.Indicator
        data-slot="progress-indicator"
        className="bg-primary h-full w-full flex-1 transition-all"
        style={{ transform: `translateX(-${100 - (value || 0)}%)` }} />
    </ProgressPrimitive.Root>
  );
}
```

**Rendered DOM**: `<div role="progressbar" aria-valuemin="0" aria-valuemax="100"
aria-valuenow="42" data-state="loading" data-value="42" data-max="100">…</div>`.
Radix sets `aria-valuenow` only when `value !== null`; pass `value={null}` to
get **indeterminate** (drops `aria-valuenow` and sets `data-state="indeterminate"`).
That's the hook for our own CSS shimmer (§2).

**Recommendation**: install it (`npx shadcn@latest add progress`). The current
hand-rolled `progress.tsx` is fine but Radix gives us indeterminate + the
`data-state` selector hook for free. We can keep our existing API surface by
re-exporting Radix's Progress as the default and not touching call sites.

Bundle cost: `@radix-ui/react-progress` is ~2 kB gzip.

## 2. Smooth animation between sparse updates

The wire ships **~3 events/sec** during `archiving_files`, each bumping by
roughly 0.17 %. Naive `width: x%` updates produce visible micro-jumps. Five
options, ranked:

| Option                                          | Effort | Smoothness | Verdict          |
|-------------------------------------------------|--------|------------|------------------|
| CSS `transition: width 300ms ease-out`          | none   | OK         | current; minimum |
| CSS `transition: transform 600ms ease-out`      | trivial| Better     | **WINNER for the bar** |
| `requestAnimationFrame` interpolation in JS     | medium | Excellent  | overkill         |
| Shimmer overlay (animated gradient)             | low    | Masks jumps| use for *indeterminate* |
| `@number-flow/react` for counter text           | low    | Excellent  | **WINNER for counters** |
| `react-spring`                                  | high   | Excellent  | overkill, +8 kB  |

**Why transform over width**: width animation reflows; transform is composited.
Combine with a **longer** transition than the inter-event interval (~600 ms vs
~330 ms) so each new value catches the bar mid-flight and continues the
animation seamlessly — gives a *continuous fill* illusion without an rAF loop.

**Counters**: install `@number-flow/react` (4 kB, MIT, framework-agnostic core).
It diffs digits and tween-animates each column — perfect for `42 → 43` chunk
counts and `12.4 MB → 12.7 MB` byte counts. Falls back to plain text under
`prefers-reduced-motion`.

```bash
pnpm --filter @wpmgr/web add @number-flow/react
```

```tsx
import NumberFlow from "@number-flow/react";
<NumberFlow value={bytesDone} format={{ notation: "compact" }} />
```

**Shimmer for indeterminate** (queued / submitting_manifest where there's no
percentage): a 1.5 s linear gradient sweep across the bar. Gated by
`motion-safe:` so reduced-motion users get a static stripe.

```css
@keyframes wpmgr-shimmer { from { transform: translateX(-100%); } to { transform: translateX(100%); } }
.shimmer { @apply motion-safe:animate-[wpmgr-shimmer_1.5s_linear_infinite]; }
```

## 3. Phase stepper component

shadcn/ui does **not** ship a Stepper in the core registry as of 2026-05.
Third-party options:

- **shadcn-stepper.vercel.app** — single file, MIT, copy-paste. Solid baseline.
- **shadcn-extension.com/stepper** — full-featured but pulls `framer-motion` (we
  don't want it).
- **Aceternity / Magic UI** — visually flashy, hover effects we don't need; both
  require Framer Motion.

**Recommendation**: **DIY in ~80 lines.** A phpbu pipeline is 5–7 ordered
phases with a closed set of states (`queued | running | completed | failed |
pending`); we don't need swipe-able forms or branching. Build it with
`<ol>` + Tailwind utilities + Lucide icons; this stays in our coding style and
adds zero deps. The hand-rolled version is `phase-stepper.tsx` in §8a.

For the **inline table row** (~280 px), skip the stepper entirely — show only
the current phase label + a 1-px-tall progress bar (the existing
`InlineSnapshotProgress` is already correct).

## 4. Lucide icons for phase states

**Important note on the installed version**: `lucide-react ^1.16` is the
legacy 0.x line (npm re-indexed the package in 2024 and tagged early
post-fork releases as `1.16.x`). The icon names below all exist in that
version; verify by importing in the IDE before committing.

| State        | Icon                  | Reasoning                          |
|--------------|-----------------------|-------------------------------------|
| `pending`    | `Circle`              | empty outline = "future"           |
| `queued`     | `Hourglass`           | waiting in line                    |
| `running`    | `Loader2` *(spin)*    | the canonical Lucide spinner       |
| `completed`  | `CircleCheck`         | unambiguous success                |
| `failed`     | `CircleX`             | matches `--color-destructive`      |
| skipped      | `MinusCircle`         | if/when we add it                  |

Use `aria-hidden` and pair with a visually hidden status word so screen
readers don't get *"check-mark-icon"* noise.

```tsx
<Loader2 aria-hidden className="size-4 animate-spin text-blue-600" />
<span className="sr-only">running</span>
```

The `animate-spin` utility respects `prefers-reduced-motion` by default in
Tailwind v4 (the keyframes are gated by the media query in the preflight).

## 5. Time / ETA calculation

Algorithm: **exponentially weighted throughput over a sliding window**, with a
*"calculating"* gate until we have >= 5 s of samples or >= 5 events.

```ts
samples: { ts: number /* ms */, pct: number /* 0..100 */ }[]
window = last 15 s of samples
throughputPctPerSec = (pct_last - pct_first_in_window) / dt_sec   // EWMA preferred
etaSeconds = (100 - pct_last) / throughputPctPerSec
isReliable = window.length >= 3 && throughputPctPerSec > 0.05
```

Edge cases:
- Throughput **drops to zero** (network stall, dedup-only burst): keep the last
  ETA but flip `isReliable=false` → render *"Still working…"*.
- **Reverses** (status flips from `uploading` back to `archiving_files`): clear
  samples on phase change; start over.
- **Sparse** (<3 samples in window): render *"Calculating…"*.

**Display rules** (matching iOS Files / YouTube convention):

| eta seconds        | label                          |
|--------------------|--------------------------------|
| n/a or unreliable  | "Calculating…"                 |
| <= 10              | "A few seconds left"           |
| <= 60              | "Less than a minute left"      |
| <= 90              | "About a minute left"          |
| <= 3600            | "About N minutes left"         |
| > 3600             | "More than an hour left"       |

React hook signature:

```ts
function useEta(samples: ProgressSample[]): {
  etaSeconds: number | null;
  throughputPctPerSec: number;
  isReliable: boolean;
  label: string;
}
```

Implemented as a `useMemo` over `samples` — no internal state, no timers.
Samples themselves are built from a ring-buffer `useRef` in the parent that
pushes on each SSE event (kept out of React state to avoid re-rendering every
~330 ms).

## 6. Reduced motion

Tailwind v4 ships **`motion-safe:`** and **`motion-reduce:`** variants
unchanged from v3. Use both:

```tsx
<div className="motion-safe:transition-transform motion-safe:duration-500 motion-reduce:transition-none" />
```

For the indeterminate shimmer, ALL of these must be `motion-safe`-gated:

- `animate-spin` on `Loader2` — already gated by Tailwind preflight ✓
- `animate-pulse` on the live/polling dot — Tailwind preflight gates it ✓
- custom `wpmgr-shimmer` keyframes — **NOT auto-gated**, must add
  `motion-safe:animate-[...]` ourselves

The number flow component reads the media query directly; nothing to do.

Add to `globals.css`:

```css
@media (prefers-reduced-motion: reduce) {
  .wpmgr-bar-fill { transition: none !important; }
}
```

## 7. Per-row vs. per-card density

Two surfaces, one data source:

| Surface                   | Width  | Components                                              |
|---------------------------|--------|---------------------------------------------------------|
| Site detail table cell    | 280 px | label + counter + 1 px bar (current `InlineSnapshotProgress`) |
| Snapshot detail page card | 100 %  | stepper + animated bar + NumberFlow counters + ETA + artifact path |

Both consume the **same** `useBackupStream(snapshotId)` hook — that hook
patches the TanStack Query cache, so any `useBackup(id)` subscriber re-renders
with fresh `snapshot.progress`. We don't lift state into a context.

Shared logic lives in a pure helper, **`formatProgress.ts`** (§8d), which takes
a `BackupSnapshot` and returns a normalised `{ phase, label, percent, counter,
bytesDone, artifact, isIndeterminate }`. Both components call it.

### a) `apps/web/src/features/backups/phase-stepper.tsx`

```tsx
import {
  Circle, CircleCheck, CircleX, Hourglass, Loader2,
} from "lucide-react";
import { cn } from "@/lib/utils";

export type PhaseStatus = "pending" | "queued" | "running" | "completed" | "failed" | "skipped";
export type Phase = { id: string; label: string; status: PhaseStatus };

const ICONS: Record<PhaseStatus, React.ComponentType<{ className?: string }>> = {
  pending: Circle,
  queued: Hourglass,
  running: Loader2,
  completed: CircleCheck,
  failed: CircleX,
  skipped: Circle,
};

const COLORS: Record<PhaseStatus, string> = {
  pending: "text-[var(--color-muted-foreground)]",
  queued: "text-amber-600 dark:text-amber-400",
  running: "text-blue-600 dark:text-blue-400",
  completed: "text-green-600 dark:text-green-400",
  failed: "text-[var(--color-destructive)]",
  skipped: "text-[var(--color-muted-foreground)]",
};

export function PhaseStepper({ phases }: { phases: Phase[] }) {
  return (
    <ol className="space-y-2" role="list" aria-label="Backup phases">
      {phases.map((p, i) => {
        const Icon = ICONS[p.status];
        const isLast = i === phases.length - 1;
        return (
          <li key={p.id} className="relative flex items-start gap-3">
            <div className="flex flex-col items-center">
              <Icon
                aria-hidden
                className={cn(
                  "size-5 shrink-0",
                  COLORS[p.status],
                  p.status === "running" && "motion-safe:animate-spin",
                )}
              />
              {!isLast && (
                <span
                  aria-hidden
                  className={cn(
                    "mt-1 h-6 w-px",
                    p.status === "completed"
                      ? "bg-green-600/40"
                      : "bg-[var(--color-border)]",
                  )}
                />
              )}
            </div>
            <div className="flex-1 pb-2">
              <p className={cn(
                "text-sm",
                p.status === "running" && "font-semibold",
                p.status === "pending" && "text-[var(--color-muted-foreground)]",
              )}>
                {p.label}
              </p>
              <span className="sr-only"> — {p.status}</span>
            </div>
          </li>
        );
      })}
    </ol>
  );
}
```

### b) `apps/web/src/features/backups/animated-progress-bar.tsx`

```tsx
import { Progress } from "@/components/ui/progress";
import { cn } from "@/lib/utils";

interface Props {
  value: number | null;          // null → indeterminate
  max?: number;
  label: string;                 // for ARIA
  className?: string;
}

export function AnimatedProgressBar({ value, max = 100, label, className }: Props) {
  const isIndeterminate = value === null;
  const pct = isIndeterminate ? 0 : Math.min(100, Math.round((value / max) * 100));

  // Use transform: translateX, transition longer than inter-event gap (~330 ms)
  // so each new value catches the in-flight animation and continues smoothly.
  return (
    <div
      role="progressbar"
      aria-valuemin={0}
      aria-valuemax={max}
      aria-valuenow={isIndeterminate ? undefined : value}
      aria-label={label}
      className={cn(
        "relative h-2 w-full overflow-hidden rounded-full bg-[var(--color-muted)]",
        className,
      )}
    >
      {isIndeterminate ? (
        <div
          aria-hidden
          className="absolute inset-y-0 left-0 w-1/3 rounded-full bg-gradient-to-r from-transparent via-[var(--color-primary)] to-transparent motion-safe:animate-[wpmgr-shimmer_1.5s_linear_infinite]"
        />
      ) : (
        <div
          aria-hidden
          className="h-full origin-left rounded-full bg-[var(--color-primary)] motion-safe:transition-transform motion-safe:duration-[600ms] motion-safe:ease-out motion-reduce:transition-none"
          style={{ transform: `translateX(${pct - 100}%)`, width: "100%" }}
        />
      )}
    </div>
  );
}
```

### c) `apps/web/src/features/backups/eta-display.tsx`

```tsx
import { useMemo } from "react";

export type ProgressSample = { ts: number; pct: number };
const WINDOW_MS = 15_000;
const MIN_SAMPLES = 3;
const MIN_THROUGHPUT = 0.05; // pct/sec

export function useEta(samples: ProgressSample[]) {
  return useMemo(() => {
    if (samples.length < MIN_SAMPLES) {
      return { etaSeconds: null, throughputPctPerSec: 0, isReliable: false, label: "Calculating…" };
    }
    const last = samples[samples.length - 1];
    const cutoff = last.ts - WINDOW_MS;
    const window = samples.filter((s) => s.ts >= cutoff);
    const first = window[0];
    const dtSec = Math.max(0.001, (last.ts - first.ts) / 1000);
    const dPct = last.pct - first.pct;
    const throughput = dPct / dtSec;
    if (throughput < MIN_THROUGHPUT) {
      return { etaSeconds: null, throughputPctPerSec: throughput, isReliable: false, label: "Still working…" };
    }
    const remaining = Math.max(0, 100 - last.pct);
    const etaSeconds = Math.round(remaining / throughput);
    return { etaSeconds, throughputPctPerSec: throughput, isReliable: true, label: formatEta(etaSeconds) };
  }, [samples]);
}

function formatEta(s: number): string {
  if (s <= 10) return "A few seconds left";
  if (s <= 60) return "Less than a minute left";
  if (s <= 90) return "About a minute left";
  if (s <= 3600) return `About ${Math.round(s / 60)} minutes left`;
  return "More than an hour left";
}

export function EtaDisplay({ samples }: { samples: ProgressSample[] }) {
  const { label, isReliable } = useEta(samples);
  return (
    <span
      className="text-xs text-[var(--color-muted-foreground)]"
      aria-live="polite"
      data-reliable={isReliable}
    >
      {label}
    </span>
  );
}
```

### d) `apps/web/src/features/backups/formatProgress.ts`

```ts
import type { BackupSnapshot } from "@wpmgr/api";

const PHASE_LABEL: Record<string, string> = {
  queued: "Queued",
  started: "Started",
  dumping_db: "Dumping database",
  archiving_files: "Archiving files",
  compressing_files: "Compressing files",
  encrypting: "Encrypting",
  uploading: "Uploading chunks",
  encrypting_uploading: "Encrypting & uploading",
  submitting_manifest: "Submitting manifest",
  completed: "Completed",
  failed: "Failed",
};

const INDETERMINATE_PHASES = new Set([
  "queued", "started", "dumping_db", "submitting_manifest",
]);

export interface FormattedProgress {
  phase: string;
  label: string;
  percent: number | null;        // null → render indeterminate
  counter: string | null;        // e.g. "42/256 chunks"
  bytesDone: number | null;
  filesDone: number | null;
  filesTotal: number | null;
  chunksDone: number | null;
  chunksTotal: number | null;
  artifact: string | null;
  currentFile: string | null;
  dedupHit: boolean;
  isIndeterminate: boolean;
}

type PhaseDetail = {
  files_done?: number; files_total?: number;
  chunks_done?: number; chunks_total?: number;
  bytes_done?: number; bytes_written?: number;
  current_file?: string; current_artifact?: string;
  artifact?: string; source?: string;
  dedup_hit?: boolean;
};

function num(v: unknown): number | null {
  return typeof v === "number" && Number.isFinite(v) ? v : null;
}

export function formatProgress(snapshot: BackupSnapshot): FormattedProgress {
  const progress = snapshot.progress as { phase?: string; phase_detail?: PhaseDetail } | null;
  const phase = progress?.phase ?? "queued";
  const d: PhaseDetail = progress?.phase_detail ?? {};

  const filesDone = num(d.files_done);
  const filesTotal = num(d.files_total);
  const chunksDone = num(d.chunks_done);
  const chunksTotal = num(d.chunks_total);
  const bytesDone = num(d.bytes_done) ?? num(d.bytes_written);

  let percent: number | null = null;
  let counter: string | null = null;
  if (filesTotal && filesTotal > 0) {
    percent = Math.min(100, Math.round(((filesDone ?? 0) / filesTotal) * 100));
    counter = `${filesDone ?? 0}/${filesTotal} files`;
  } else if (chunksTotal && chunksTotal > 0) {
    percent = Math.min(100, Math.round(((chunksDone ?? 0) / chunksTotal) * 100));
    counter = `${chunksDone ?? 0}/${chunksTotal} chunks`;
  }

  const isIndeterminate = percent === null || INDETERMINATE_PHASES.has(phase);

  return {
    phase,
    label: PHASE_LABEL[phase] ?? phase,
    percent,
    counter,
    bytesDone,
    filesDone, filesTotal, chunksDone, chunksTotal,
    artifact: d.current_artifact ?? d.artifact ?? null,
    currentFile: d.current_file ?? null,
    dedupHit: d.dedup_hit === true,
    isIndeterminate,
  };
}
```
